# SECURITY.md

## Поточний стан

Цей документ описує безпеку в межах того, що реально реалізовано в поточному репозиторії.

---

## Доступ до файлової системи книг

Каталог книг:

- `/app/static/books` у контейнері
- `./static/books` на хості при поточному `docker-compose.yml`

Для маршрутів:

- `/api/folder`
- `/book/<path>`

використовується перевірка шляху через `resolve()/commonpath`, щоб не допустити вихід за межі каталогу книг.

Це блокує path traversal на кшталт `../...` при зверненні до дерева книг і при віддачі HTML-файлів.

---

## HTTP surface

У поточному compose-розгортанні публічним endpoint є `nginx`, який слухає порт `80` і проксіює трафік у `gunicorn`.

При локальному запуску через `python app.py` застосунок також може бути доступний напряму Flask dev server без `nginx`.

---

## Docker security

У наявних файлах репозиторію:

- немає `privileged: true`
- немає монтування Docker socket

Ці обмеження варто зберегти і надалі.

---

## Nginx

`nginx`:

- є публічним HTTP endpoint у compose-режимі
- віддає `/static/` безпосередньо
- проксіює інші запити в `gunicorn`

---

## File permissions

Конкретна схема прав доступу на хості залежить від середовища розгортання.

У поточному репозиторії окрема схема `uid/gid` не зафіксована, тому це потрібно контролювати на рівні хоста або контейнерного середовища.

---

## Network

Поточні порти:

- `80` через `nginx` у Docker Compose
- `5000` всередині internal compose network для `app`

---

## Updates

Рекомендується регулярно оновлювати:

- Docker base image
- Python dependencies
- `nginx` image
- Flask application code

---

## Backup policy

Каталог книг повинен регулярно резервуватися.

У поточній конфігурації це:

- `./static/books`
